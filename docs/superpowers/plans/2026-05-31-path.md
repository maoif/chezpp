# Path Library Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add the `(chezpp path)` library for pure Unix and Windows path manipulation.

**Architecture:** Represent paths as opaque records containing a flavor, root kind, root payload, component list, and trailing-directory marker. Keep filesystem I/O out of the library except for explicit current-user home lookup in `path-expand-user`. Export object-oriented path operations and re-export them through `(chezpp)`.

**Tech Stack:** ChezScheme libraries, Chezpp `pcheck`, existing `mat` tests, `make test-some`.
